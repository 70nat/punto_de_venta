# Modelo Híbrido: Cascada + Scrum aplicado al Proyecto de Punto de Venta en C++

## 1. Introducción
El proyecto consiste en el desarrollo de un sistema de Punto de Venta (POS) en C++ utilizando Xcode. Para su gestión se propone un modelo híbrido que combina las metodologías **Cascada** y **Scrum**, aprovechando la estructura secuencial del primero y la flexibilidad iterativa del segundo.

## 2. Justificación de la elección del modelo híbrido

### Por qué Cascada
- Define un orden lógico y rígido al inicio.
- Permite documentar requisitos y arquitectura del sistema.
- Es ideal para la planificación inicial del POS.

### Por qué Scrum
- Permite avanzar por iteraciones cortas (sprints).
- Flexibilidad para corregir y mejorar módulos.
- Facilita evidencia continua en GitHub.

### Por qué combinarlas
- Cascada da claridad inicial.
- Scrum permite mejoras continuas.
- Juntas reducen riesgos y mantienen orden + agilidad.

## 3. Aportes de cada metodología

### Aportes de Cascada
- Requisitos claros del POS.
- Diseño completo de la arquitectura.
- Definición de módulos (productos, ventas, usuarios).

### Aportes de Scrum
- Incrementos funcionales semanales.
- Priorización de tareas mediante backlog.
- Entregables continuos.

## 4. Aplicación del modelo híbrido al proyecto

### Fase 1 – Análisis (Cascada)
- Requisitos del POS.
- Definición de módulos principales.

### Fase 2 – Diseño (Cascada)
- Diseño de clases en C++.
- Estructura del flujo de ventas, inventario y reportes.

### Fase 3 – Implementación Iterativa (Scrum)

#### Sprint 1
- CRUD de productos.

#### Sprint 2
- Registro de ventas y actualización de inventario.

#### Sprint 3
- Módulo de usuarios y reportes básicos.

#### Sprint 4
- Pruebas, mejoras, optimización.

### Fase 4 – Pruebas finales (Cascada)
- Evaluación completa del sistema.

### Fase 5 – Documentación final
- Reporte de funcionamiento.
- Evidencias en GitHub.

## 5. Ventajas del modelo híbrido
- Claridad + flexibilidad.
- Mejor control del tiempo.
- Reducción de riesgos.
- Entregables constantes.

## 6. Conclusión
El modelo híbrido Cascada + Scrum permite un desarrollo ordenado, adaptable y con entregables funcionales periódicos, ideal para el proyecto de Punto de Venta.
